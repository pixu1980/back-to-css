Accessibility hooks ride along attributes

If attributes drive state, the same values can be announced via live regions and referenced by aria-* relationships. 
        <div role="status" aria-live="polite" data-speed="88" id="speed-status"></div>
<button aria-controls="speed-status" data-speed="88">Jump</button>
        [role='status'][data-speed]::before {
  content: 'Speed ' attr(data-speed) ' mph';
}

button[data-speed][aria-pressed='true'] {
  outline-color: var(--btf-teal);
}

Assistive tech hears exactly what the CSS sees.
Live regions mirror style logic perfectly because attributes are shared.
