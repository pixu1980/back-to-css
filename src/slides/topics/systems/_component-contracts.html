Document attribute contracts like APIs

Each component declares which attributes it understands, their types, and the CSS logic they feed. No hidden class magic. 
        <flux-panel
  data-era="future"
  data-speed="88"
  data-energy="1.21"
  aria-live="polite"
></flux-panel>
        @scope (flux-panel[data-speed][data-energy]) {
  :scope {
    --flux-speed: attr(data-speed number);
    --flux-energy: attr(data-energy number);
  }

  .status::after {
    content: if(
      attr(data-speed number) >= 88,
      'Brace for jump',
      'Hold course'
    );
  }
}
  • data-eratoken: past|future
  • data-speednumber mph
  • data-energynumber GW
