HTML as the State Layer

Attributes become your single source of truth. CSS reads, never writes.

The attribute bus pattern

    
    <!-- All state lives in HTML -->
    <article
      class="dashboard"
      data-speed="88"
      data-energy="1.21"
      data-era="future"
      data-status="ready">
      ...
    </article>
    
  
    
    /* CSS reads state, applies logic */
    .dashboard {
      --speed: attr(data-speed number, 0);
      --energy: attr(data-energy number, 0);
      --is-future: attr(data-era token) = future;
      --is-ready: attr(data-status token) = ready;
    }

No side channel classes

Before: class juggling
        
        <div class="card
          speed-88
          energy-high
          theme-dark
          status-active">
Classes encode state but get messy fast.
After: clean attributes
        
        <div class="card"
          data-speed="88"
          data-energy="1.21"
          data-theme="dark"
          data-status="active">
Attributes are semantic and typed.
