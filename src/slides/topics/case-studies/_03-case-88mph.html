Case: 88 mph trigger without JS logic

We shipped a safety checklist where CSS decided when the UI could fire the time jump. 
        
        .jump-panel::after {
          content: if(attr(data-speed number) >= 88, 'Temporal jump ready', 'Keep accelerating');
        }

        .jump-panel button[disabled] {
          opacity: 0.4;
        }

Jump disabled until 88 mph. CSS alone enforces it.
QA loved it: no JS branch to miss, and screen readers hear the same state string.
